# 特价机票发现平台 PRD 文档
**产品名称：** FlightSnap · 特价机票发现平台  
**版本：** V1.0  
**文档日期：** 2026-04-10  

---

## 一、产品概述

### 1.1 背景与问题

| 用户痛点 | 具体表现 |
|---------|---------|
| 信息分散 | 携程、去哪儿、飞猪、各航司App 各自推送，用户需多平台比对 |
| 时机难把握 | 特价票放出时间随机，稍纵即逝，用户常常错过 |
| 规则复杂 | 退改签限制、行李额、中转时长难以快速判断性价比 |
| 决策成本高 | 缺少横向对比，不知道"这个价格值不值" |
| 通知干扰多 | 各平台推送泛滥，与用户真实需求不匹配 |

### 1.2 产品定位

> 做用户的"机票雷达"——**精准发现 + 智能提醒 + 一键比价**，帮助对价格敏感的旅行者以最低成本捕捉特价机票。

### 1.3 目标用户

- **核心用户：** 25–40 岁，有自由行习惯，价格敏感，出行弹性较大
- **次要用户：** 商旅族（需要寻找经济舱特价）、学生群体（长假出游预算有限）

## 二、产品目标与成功指标

### 2.1 产品目标（V1.0）

1. 聚合主流OTA及航司特价信息，实现"一站发现"
2. 支持用户设置价格预警，实现"精准触达"
3. 提供结构化的性价比评分，降低用户决策成本

### 2.2 关键指标 (KPI)

| 指标类型 | 指标名称 | V1.0 目标 |
|---------|---------|---------|
| 用户规模 | 月活用户(MAU) | 5万 |
| 用户粘性 | 7日留存率 | ≥ 30% |
| 核心行为 | 价格预警设置率 | ≥ 40%（注册用户） |
| 转化 | 点击跳转购买率 | ≥ 15% |
| 内容质量 | 特价票准确率 | ≥ 95% |

## 三、功能架构

### 3.1 整体信息架构

```
FlightSnap
├── 首页（发现）
│   ├── 热门特价 Feed
│   ├── 实时价格日历
│   └── 目的地灵感推荐
├── 搜索
│   ├── 出发地 / 目的地 / 日期
│   ├── 价格走势图
│   └── 多平台比价列表
├── 雷达（价格预警）
│   ├── 我的预警设置
│   ├── 触发历史
│   └── 新建预警
├── 发现（内容）
│   ├── 特价专题（如"五一出行精选"）
│   ├── 目的地攻略卡片
│   └── 用户晒单
└── 我的
    ├── 关注路线
    ├── 历史查看
    └── 账号设置
```

### 3.2 核心功能清单

#### 模块 A：智能特价发现（优先级 P0）

| 功能 | 描述 | 优先级 |
|-----|-----|-------|
| 特价 Feed 流 | 聚合各平台当日特价，按性价比排序展示 | P0 |
| 价格日历视图 | 以月历形式展示全月最低价，颜色热力标注 | P0 |
| 低价预测标签 | 基于历史数据标注"历史最低"/"近期最低" | P0 |
| 灵感目的地 | 用户不限目的地，展示"从出发地出发最便宜去哪" | P1 |

#### 模块 B：价格预警雷达（优先级 P0）

| 功能 | 描述 | 优先级 |
|-----|-----|-------|
| 路线预警 | 设置出发地→目的地 + 期望价格阈值 | P0 |
| 灵活日期预警 | 支持设置"任意周末"或"±3天弹性" | P0 |
| 多渠道推送 | 微信/邮件/站内消息三选一或叠加 | P0 |
| 预警历史 | 查看历史触发记录及当时价格快照 | P1 |

#### 模块 C：多平台比价（优先级 P1）

| 功能 | 描述 | 优先级 |
|-----|-----|-------|
| 聚合搜索 | 统一搜索界面，同步返回各渠道价格 | P1 |
| 综合评分 | 综合价格/行李额/退改签/飞行时长给出性价比分 | P1 |
| 一键跳转 | 点击直达对应平台购买页（Deep Link） | P1 |
| 历史价格曲线 | 展示该航班/路线近30天价格走势 | P1 |

#### 模块 D：内容与社区（优先级 P2）

| 功能 | 描述 | 优先级 |
|-----|-----|-------|
| 特价专题 | 运营策划的主题页（如"黄金周抢票攻略"） | P2 |
| 低价晒单 | 用户分享抢到的特价票截图，形成UGC | P2 |
| 目的地卡片 | 简化版目的地介绍，引导购票 | P2 |

---

### 4.2 价格日历视图

**设计说明：**
- 以月份为单位展示，每个日期格显示当日最低价
- 颜色编码：深绿（历史最低区间）→ 黄（普通价）→ 红（高峰溢价）
- 支持左右滑动切换月份，最多查看未来3个月
- 点击具体日期，展示当日所有可用航班及各平台价格

---

### 4.3 雷达预警设置页

**表单字段：**
```
出发城市：[北京 ▼]
目的地：  [三亚 ▼]（可选"任意目的地"）
出行日期：[日期选择] / [弹性±3天] / [任意周末]
目标价格：[¥ ____]（系统自动建议当前最低价-20%）
推送方式：[✓微信] [□邮件] [□App]
有效期：  [30天 ▼]

[创建预警]
```

**智能辅助：**
- 系统根据历史数据自动建议合理目标价
- 显示"该路线近期最低价 ¥XXX，你的目标 ¥XXX 触发概率约 XX%"

## 五、数据架构与技术要点（概念层）

### 5.1 数据来源策略

| 来源类型 | 获取方式 | 示例 |
|---------|---------|------|
| OTA 官方合作 | API 接入或联盟推广接口 | 携程开放平台、去哪儿联盟 |
| 航司直营 | 官网价格爬取（授权）/ 官方API | 国航、东航、南航 |
| 用户UGC | 用户主动上传票价截图 | 低价晒单功能 |

### 5.2 性价比评分算法（示意）

```
综合评分 = 
  价格得分（40%）× [1 - 当前价 / 历史均价]
+ 便利得分（30%）× [行李额 + 退改签宽松度]
+ 时效得分（20%）× [飞行时长 + 中转次数]
+ 准点得分（10%）× [航班历史准点率]
```

## 七、竞品分析

| 产品 | 优势 | 劣势 | FlightSnap 差异化 |
|-----|-----|------|-----------------|
| 航班管家 | 用户基础大，数据全 | 以管理为主，发现弱 | 强化"发现"和"性价比推荐" |
| 去哪儿 | 比价能力强 | 平台导向，广告多 | 中立立场，性价比评分透明 |
| Google Flights | 价格日历好用 | 国内数据弱 | 聚焦国内+周边国际，落地更准 |
| 微信机票特价群 | 信息及时 | 无结构，难筛选 | 结构化聚合，智能匹配 |

## 九、风险与对策

| 风险 | 等级 | 对策 |
|-----|-----|------|
| 数据授权获取难 | 高 | 优先接入有开放联盟的平台；MVP阶段可用人工维护数据 |
| 价格数据实时性 | 中 | 前端标注数据更新时间，引导用户跳转平台确认 |
| 流量依赖第三方跳转 | 中 | 构建用户预警黏性，降低对单次跳转的依赖 |
| 合规与版权 | 中 | 仅展示价格信息（非完整商品页），添加来源标注 |
